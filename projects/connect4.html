Connect 4.

How it works:

Single Player:

this sends requests to a server. the server returns the best move, given a position as a url parameter.

a position is generated as a string of numbers, with each number representing a column on the board.

the server returns an object with each of the possible moves, and gives each move a score.

this uses the min-max algorithm, which is used for board games. it calculates each of the moves, and gives each move a score based on how good it is.

Online Multiplayer:

you can play with another player online. this uses firebase's real time database.

this was definitely a more challenging thing to build, but i'm happy with the result.

this was also filled with bugs. i ended up squashing them. it was not fun.

Two Player:

this is a standard connect 4 game you can play with someone sitting next to you.

nothing new or flashy about this.
